## Branch `fix/gpu-duplicate-shares` — CUDA re-submits the same share (dup rejects)

**Symptom:** a slow GPU (RTX 2080, vardiff diff 2-4) showed ~10% rejected
shares, ALL code 22 "duplicate share". The vast RTX 5090s (diff 35) showed
~0%. Direct log evidence: the same `(job, xn2, nonce)` triple submitted 2-3x.

**Root cause (`src/backends/cuda.rs` `hash_range`):** the two pipes' device
buffers (`found_flag`, `found_nonce`) are persistent on `self` across calls.
On a find the old code did `if let Some(res) = drain_pipe(pipe) { return
Some(res); }` — returning BEFORE `pipe.in_flight = false`. So the pipe stayed
`in_flight=true` with its found buffer intact, and the NEXT hash_range call
re-drained the same nonce (before any new launch zeroed found_flag) and
re-submitted it. The sibling pipe's abandoned launch leaked the same way.

**Why only low difficulty:** at high diff most chunks find nothing, so the
`drain -> None -> in_flight=false` reset path runs every launch and clears the
leak. At low diff nearly every chunk finds, so the early-return path (which
skipped the reset) was taken constantly. Fast cards never saw it; the long
tail of CPU/low-end-GPU miners all would have — bad launch optics + support
load, so worth fixing before going public.

**Fix:** mirror the OpenCL backend, which was already correct — reset
`in_flight = false` before using the drained result, and drain+reset the
sibling pipe before returning. ~15 lines, no behavior change on the happy path.

**Verified on the actual repro hardware** (mats' RTX 2080, same low diff):
- before: 19 rejected / 180 accepted (~10%), all duplicates
- after: 1 rejected / 62 accepted (~1.6%), 1 duplicate line in 12 min
Duplicate rate collapsed to near-zero. 70/70 tests pass, clean build. The
residual single dup is likely a rare timing edge or noise, not the same bug.

The fixed binary is running live on mats' machine
(`~/.local/bin/cairn-miner-fixed`, still cairn-miner/0.2.2) so the pool has
launch hashrate. Fold this into the next release (v0.3.0 per inverse's plan,
or a v0.2.3 if launch wants it sooner).
