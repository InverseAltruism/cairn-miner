# Worklog

Shared session log between the pool operators (mats + inverse), this repo's
slice of the project. One file per work session, `YYYY-MM-DD-<author>.md`,
newest entry = current state. Sibling repos (cairn-pool, the node fork) have
their own `worklog/`.

Purpose: we both run AI-assisted sessions. Start a session with "read
worklog/ and catch up" so the AI knows what the other party did since last
time: decisions, branches in flight, open questions.

Each entry: what (branches/commits), why (non-obvious reasoning), state
(merged/pushed/deployed/uncommitted), open questions for the other party.
